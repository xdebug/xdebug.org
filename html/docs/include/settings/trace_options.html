This settings accepts a bitfield to enable options:

0x01
Trace file data will be appended to an already existing file with the same name, instead of it being overwritten.
0x02
Switches the file format to a tab separated format. The format is described in the [CFG:trace_format] setting as "format 1".
0x04
Switches to a file format that shows data as an HTML table
0x08
With this bit set, .xt is not added automatically to the end of trace file names.
0x10
Switches to a file format that provides information for "cost" flamegraphs.
0x20
Switches to a file format that provides information for "memory usage" flamegraphs.

To combine multiple flags, you can use bitwise-OR (|).

xdebug.trace_options=0x02|0x08 enables both the tab separated format, and stops the addition of .xt to the end of the file name.
